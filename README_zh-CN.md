## 📌 先决条件

**您必须在车辆上拥有有效的FSD套餐——无论是购买的还是订阅的。** 此主板可在CAN总线级别启用FSD功能，但车辆仍需具备来自特斯拉的有效FSD授权。

如果您的地区无法订阅FSD，您可以采取以下方法绕过此限制：
1、创建一个FSD订阅服务可以使用地区的特斯拉账号（例如：加拿大、美国）
2、将车辆转移到该账号下
3、在该账号下订阅或购买FSD服务

这使您能够从世界任何地方激活FSD订阅。

## 🛠️ 它的作用

此固件运行于Adafruit CAN总线 FeatherWing（基于MCP25625/MCP2515）。它会拦截特定的CAN总线消息，以启用并配置全自动驾驶（FSD）。此外，ASS（实际智能召唤）已不再受欧盟法规限制。 中国大陆相关法规情况请自行查询。

🚗 核心功能
- 拦截特定的 CAN 总线消息
- 将其重新传输到车辆总线上


🧠 FSD 激活逻辑
- 监听与 Autopilot（自动辅助驾驶）相关的 CAN数据 帧
- 检查车辆 Autopilot 设置中是否启用了“交通信号灯和停车标志控制”
- 使用此设置作为“完全自动驾驶”（FSD）的触发器
- 调整 CAN 消息中的必要校验位以激活 FSD


⚙️ 附加行为
- 读取跟车距离拨杆设置
- 动态将其映射到速度配置文件（Speed Profile）


⚙️ HW4 - FSD V14 功能
- 接近紧急车辆检测（Approaching Emergency Vehicle Detection）

### 支持的硬件型号

通过#define HW指令，在RP2040CAN.ino中选择您的硬件：

| 版本 | 版本标识 | 监听串口标识 | 说明 |
|----------|------------------|---------------------|-------|
| `LEGACY` | HW3 Retrofit     | 1006, 69            | 设置 FSD 启用位，并通过跟车距离控制速度配置文件 |
| `HW3`    | HW3 vehicles     | 1016, 1021          | 功能与 LEGACY 相同 |
| `HW4`    | HW4 vehicles     | 1016, 1021          | 扩展了速度配置文件范围（5 个等级） |

> **注意：** 固件版本为**2026.2.9.X**的HW4车辆搭载的是**FSD v14** 。不过，位于**2026.8.X** 分支的版本仍使用**FSD v13** 。如果您的车辆运行的是**FSD v13** （包括2026.8.X分支或更早的2026.2.9版本），即使您的车辆配备了HW4硬件，也请使用`HW3`进行编译。

### 如何确定您的硬件型号

- **Legacy（旧版）**  —— 你的车辆拥有**纵向的中控屏幕 **且为 **HW3** 。适用于最初配备或改装了 HW3 的旧款（Palladium 之前）Model S 和 Model X 车辆。
- **HW3** —— 你的车辆拥有 **横向的中控屏幕** 且为 ** HW3** 。你可以在车辆触摸屏的 **控制 → 软件 → 车辆附加信息**  中查看硬件版本。
- **HW4** —— 同上，但“车辆附加信息”屏幕显示为 **HW4** 。

### 关键行为

- **FSD 启用位**：当车辆 Autopilot 设置中启用了 **“交通信号灯和停车标志控制”** 时设置。
- **速度配置文件**：源自滚轮偏移量或跟车距离设置。
- **提示抑制**：清除“手握方向盘”提示位。
- 调试输出：当` enablePrint` 为 `true` 时，以 115200 波特率通过串口打印输出。

## 硬件要求

- Adafruit Feather M4 CAN（或带有 MCP25625/MCP2515 的兼容板）
- 电路板必须引出以下引脚（在代码顶部定义）：
	- `PIN_CAN_CS` —— MCP2515 的 SPI 片选
	- `PIN_CAN_INTERRUPT` —— 中断引脚（未使用；轮询模式）
	- `PIN_CAN_STANDBY` —— CAN 收发器待机控制
	- `PIN_CAN_RESET` —— MCP2515 硬件复位
- 连接到车辆的 CAN 总线（500 kbit/s）

### 安装

### 1. 安装Arduino IDE

从[https://www.arduino.cc/en/software](https://www.arduino.cc/en/software)下载。 

### 2. 添加Adafruit开发板软件包

1、打开 文件 → 首选项。
2、在 附加开发板管理器网址 中添加：
```
https://github.com/earlephilhower/arduino-pico/releases/download/global/package_rp2040_index.json
```
3、进入** 工具 → 开发板 → 开发板管理器**，搜索** Raspberry PI Pico/RP2040**（或你的 Feather 对应的系列）并安装。
4、选择 **Adafruit Feather RP2040 CAN** 作为开发板。

### 3. 安装所需库

通过**Sketch → 包含库 → 管理库…**或Arduino库管理器安装以下库：

-  **MCP2515** by autowp — CAN controller driver (`mcp2515.h`)

### 4. 选择您的硬件目标

在`RP2040CAN.ino`文件的顶部附近，将`HW`定义更改为与您的车辆相匹配：

```cpp
#define HW HW3  // 根据目标车辆版本自行配置为： LEGACY, HW3, 或 HW4
```

### 5. 上传

1、通过 USB 连接 Feather。
2、在**工具** 下选择正确的开发板和端口。
3、点击**上传** 。

### 6. 接线

推荐的连接点是[X179连接器](https://service.tesla.com/docs/Model3/ElectricalReference/prog-233/connector/x179/)：

| 引脚 | 信号  |
| ---- | ----- |
| 13   | CAN-H |
| 14   | CAN-L |

将 Feather 的 CAN-H 和 CAN-L 线连接到 X179 连接器的引脚 13 和 14。

对于**旧款Model 3（2020年及更早型号）** ，推荐的连接点为[X652连接器](https://service.tesla.com/docs/Model3/ElectricalReference/prog-187/connector/x652/)，前提是车辆未配备X179端口（具体取决于生产日期）：
| 引脚 | 信号 |
|-----|--------|
| 1  | CAN-H  |
| 2  | CAN-L  |

将 Feather 的 CAN-H 和 CAN-L 线缆连接到 X652 连接器的引脚 1 和 2。 


**重要提示：** 请剪掉 Feather CAN 板上的内置 120 Ω 终端电阻。车辆的 CAN 总线已自带终端电阻，再加装第二个电阻会导致通信错误。 

## 速度曲线

速度曲线控制着车辆在FSD模式下行驶的激进程度。其配置会因硬件型号的不同而有所差异：

###  传统、HW3 和 HW4 配置文件

| Distance | Profile (HW3) | Profile (HW4) |
| :--- | :--- | :--- |
| 2 | ⚡ 快速 | 🔥 最快 |
| 3 | 🟢 正常 | ⚡ 快速 |
| 4 | ❄️ 缓慢 | 🟢 正常 |
| 5 | — | ❄️ 缓慢 |
| 6 | — | 🐢 龟速 |


## 串行监视器

以**115200波特率** 打开串口监视器，以查看实时调试输出，其中显示了FSD状态和当前激活的速度曲线。通过将`enablePrint = false`来禁用日志记录。
